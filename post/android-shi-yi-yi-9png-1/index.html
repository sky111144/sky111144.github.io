Android 拾遗（一）：9.png 图片制作

2019-10-29

Android 开发中，9.png 图片是重要一环，以较小的图片尺寸，提供了自适应图片，适配不同手机分辨率，进行拉伸或压缩，减少了适配代码，也有较好的用户体验。

实验环境

  • macOS Hign Sierra，10.13.6
  • JAVA SDK：1.8.0_171
  • Android Studio：3.1.2
  • Android API：Android 6.0，Marshmallow，API 23

对比：普通 png 图片和 9.png 图片

  • 命名差异：9.png 图片 本质上还是 png 图片，只是在文件命名上标记上 9.png，以区别于普通 png图片，在安卓开发中用于适配工作。
  • 内容差异：9.png 图片 在图片上下左右四边缘，绘制四条黑色直线，用于确定 9.png 图片 的内容范围，以及拉伸程度。

制作：Android Studio 制作 9.png 图片

  • 准备好待制作的 png 图片，重命名为 xxx.9.png，放入 drawable 文件夹下。在 Android Studio 中，双击打开 xxx.9.png 会显示 9.png 图片编辑器。
  • 鼠标移动到图片的上、下、左、右四个方向的边缘，等到鼠标箭头编程可拖动状态，分别拖动编辑出四条黑线。

细节：9.png 图片的上下左右的黑线作用

拉伸区域控制
  • 左边黑线：纵向拉伸区域，必须要画的，控制纵向拉伸区域。
  • 上边黑线：横向拉伸区域，必须要画的，控制横向拉伸区域。
内容区域控制
  • 右边黑线：可选，纵向内容显示区域
  • 下边黑线：可选，横向内容显示区域
