前端动画：对比六种解决方案的差异

2019-10-11

前端动画解决方案，致力于通过动画更加优化的表达网页信息，是一个成型的特定技术领域。

简述

前端动画解决方案，按照有无数据封装协议划分为两大类，每类三种，总共六种。

数据封装协议
  1. Gif 动图
  2. Video 视频动画
  3. SVG 动画
无数据封装协议
  1. CSS 动画
  2. JavaScript + CSS 动画
  3. JavaScript + Canvas/WebGL 动画

数据封装协议类

涉及到的协议：
1、图片协议：GIF
2、视频协议：MP4、Ogg、WebM
3、图形语言：SVG、XML
方案一：GIF 动画
GIF 是一种特殊的图片格式，能够封装多张图片，以逐帧的方式，单次或循环地，自动播放动画。
  1. 体积大小：同等帧数动画，体积较大。GIF 格式本身仅仅进行数据封装，格式本身没有做压缩工作的。如果需要压缩，需要使用压缩软件压缩 GIF 文件，或者在制作的时候压缩各个图片帧。
  2. 边下边播：不能边下边播。浏览器需要一次性加载完毕所有 GIF 包含的数据，才能进行播放。
  3. 自动播放：自动播放。无法接受 JavaScript 的控制。
  4. 循环播放：制作 GIF 的时候，才能选择是否循环播放。
  5. 音频播放：无法播放音频。
  6. 制作难度：不需要编程，掌握简单的图片编辑技能，需要具备简单的动画制作技术。
  7. 制作用途：位图动画。
方案二：视频动画
前端视频动画，主要使用 video 组件播放 mp4 动画视频，通过图像算法压缩图片，进行数据封装，以逐帧的方式，单次或循环地播放动画视频。
  1. 体积大小：同等帧数动画，体积较小。使用 H264 图像算法压缩图片，质量较高，压缩比大。
  2. 边下边播：支持边下边播。
  3. 自动播放：浏览器对视频自动播放的支持程度不统一。PC 浏览器支持视频自动播放；Mobile 浏览器原生不支持视频自动播放，只有专门从内核层面定制配置才能支持自动播放。
  4. 循环播放：浏览器对视频循环播放的支持程度不统一。PC 浏览器支持视频循环播放；Mobile 浏览器原生不支持视频循环播放，只有专门从内核层面定制配置才能支持循环播放。
  5. 音频播放：可以播放音频。
  6. 制作难度：不需要编程，需要掌握摄影机、专门的动画制作软件，需要具备专门的动画制作技术。
  7. 制作用途：位图动画。
方案三：SVG 动画
SVG 动画，主要使用 XML 语法，层次清晰。
  1. 体积大小：同等帧数矢量动画，体积最小。
  2. 边下边播：不支持边下边播。
  3. 自动播放：可自动播放。
  4. 循环播放：可循环播放。
  5. 音频播放：无法播放音频。
  6. 制作难度：需要编程，需要了解 svg 语法。
  7. 制作用途：矢量动画。

无数据封装协议

涉及到的编程接口：
1、样式语言：CSS
2、DOM接口：DOM
3、图形引擎：Canvas、WebGL
方案三：CSS 动画
  1. 编程方式：声明式。
  2. 编程接口：CSS。
  3. 绘制效率：最高。
  4. 编程难度：最低。
  5. 应用范围：无交互需求的动画。
方案五：JavaScript + CSS 动画
  1. 编程方式：命令式。
  2. 编程接口：DOM。
  3. 绘制效率：最低。
  4. 编程难度：次高。
  5. 应用范围：有较简单交互需求的动画。
方案六：JavaScript + Canvas/WebGL 动画
  1. 编程方式：命令式。
  2. 编程接口：Canvas、WebGL。
  3. 绘制效率：次高。
  4. 编程难度：最高。
  5. 应用范围：有较复杂交互需求的动画。
