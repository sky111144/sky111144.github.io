前端调试利器（一）：Charles

2019-10-22

Charles 作为 HTTP 协议的代理工具，支持请求代理、数据劫持，功能非常强大。

调试准备工作

PC端
  1. 打开 Charles 的 Proxy 菜单，选中 macOS Proxy 或者 Mozilla Firefox Proxy，开始监听网络数据。

移动端

  1. 手机和PC保持连接同一个 Wi-Fi 网络。
  2. 手机将网络代理设置为PC的网络 IP，端口设置为 8888（Charles 设置的端口）。
  3. 手机浏览器访问站点 chls.pro/ssl，下载并安装 Charles 证书，用于解析 https 数据。

本地文件映射

使用场景
  • 开发者无法拿到需要调试页面的源代码
  • 开发者的本地环境和线上环境不一致。
使用方法
  1. 设定 Map local 规则



  2. 修改本地文件用于调试

网络请求映射

使用场景
  • 开发者需要将同一个网络请求，转发给不同的服务器
使用方法
  1. 打开 Charles 的 Tools 菜单，选中 Rewrite。
  2. 新建 Rewrite 规则

网络数据节流

应用场景
  • 开发者需要模拟不同网速情况下的应用。
使用方法
  1. 打开 Charles 的 Proxy 菜单，选中 Throttle Settings。
  2. 从下拉菜单 Throttle preset中，选中需要的模式，根据需要调整网络配置。
